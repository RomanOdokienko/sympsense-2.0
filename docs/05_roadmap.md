# Текущий план (без фаз)

## Направление A: Batch processing

- Разбирать входящие пакеты документов в агент-сессии.
- Начинать с 3-5 файлов и согласовывать типизацию.
- Затем проходить весь batch.

## Направление B: Реестр и UI

- Держать единый `documents` реестр.
- Отразить в UI понятный сценарий: тип, дата, статус, уверенность, связанные факты.

## Направление C: Преемственность извлечения

- Поддерживать стабильную структуру `Document Envelope + Domain Facts`.
- Эволюционно улучшать извлечение под новые форматы документов.
- Автоматизировать только то, что уже повторяется и мешает скорости.
