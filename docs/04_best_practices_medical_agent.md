# Best Practices (текущий режим)

1. Сначала реальные документы, потом инженерные улучшения.
2. Не усложнять систему раньше времени.
3. Вести единый реестр документов как опорную точку UI.
4. Помечать низкую уверенность явно (`needs_review`).
5. Нормализовывать единицы/термины постепенно, по реальным кейсам.
6. Каждое извлечение делать трассируемым к источнику.
7. Обновлять документы проекта сразу после смены договоренностей.
