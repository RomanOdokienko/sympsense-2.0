# Analyst Prompt v1 — Аналитика здоровья

Этот файл — инструкция для Claude для запуска глубокого анализа медицинской базы.
Запускается вручную раз в ~полгода или после значимого пополнения базы.

---

## Шаг 1 — Читай данные

Прочитай следующие файлы (если существуют):

```
data/canonical/facts/lab_results_v1.ndjson
data/canonical/facts/clinical_findings_v1.ndjson
data/canonical/facts/condition_mentions_v1.ndjson
data/canonical/facts/condition_clusters_v1.ndjson
data/canonical/facts/body_snapshot_v1.json
data/derived/reports/analyst_findings_v1_*.json   ← предыдущие отчёты (для сравнения)
```

## Шаг 2 — Ищи паттерны по пяти категориям

### A. Системные паттерны
Одна система тела повреждается много раз в разное время — есть ли единая причина?
Смотри: суставы, сосуды, нервная система, ЖКТ, кожа.

### B. Лабораторные аномалии
- Значение хронически выше или ниже нормы в двух и более измерениях?
- Есть тренд (растёт / падает)?
- Важный маркер не проверялся более года?
- Аномалия была зафиксирована, но никакого последующего действия не видно?

### C. Диагностические пробелы
Диагноз X поставлен → очевидный следующий шаг Y так и не был сделан.
Пример: повышенный ЛПНП → кардиолог/терапевт по этому поводу не упоминается.

### D. Кросс-доменные корреляции
Лабораторная аномалия + клиническая картина, которые могут быть связаны между собой,
но лечились у разных специалистов как независимые проблемы.

### E. Закрытые/открытые вопросы
Сравни с предыдущим отчётом:
- Находка из прошлого отчёта → появились новые данные? Вопрос закрыт или нет?
- Если никаких новых данных нет — помечай как "без изменений", не пересказывай заново.

---

## Шаг 3 — Правила качества

**Включать только если:**
- Паттерн подтверждён данными из ≥ 2 документов с разными датами, ИЛИ
- Это явный пробел (важный маркер отсутствует > 1 года)

**Не включать:**
- Разовые наблюдения без подтверждения
- То что уже есть в предыдущем отчёте без новых данных
- Общие медицинские рекомендации без привязки к конкретным данным пациента

## Шаг 4 — Формат вывода

Пиши JSON в файл `data/derived/reports/analyst_findings_v1_YYYYMMDD.json`:

```json
{
  "version": "analyst_findings_v1",
  "generated_at": "<ISO timestamp>",
  "as_of_date": "<YYYY-MM-DD>",
  "source": "claude_analysis_session",
  "summary": {
    "total_findings": <N>,
    "high_priority": <N>,
    "medium_priority": <N>,
    "low_priority": <N>,
    "key_message": "<одно предложение: главный вывод этого прогона>"
  },
  "findings": [
    {
      "id": "finding_001",
      "title": "<краткое название паттерна>",
      "priority": "high | medium | low",
      "status": "new | unchanged | updated | resolved",
      "pattern": "<что наблюдается, с датами и значениями>",
      "supporting_docs": ["<дата + описание>", ...],
      "hypothesis": "<медицинская гипотеза что это может означать>",
      "action": "<конкретный следующий шаг>"
    }
  ],
  "resolved_since_last": [
    {
      "id": "<id из предыдущего отчёта>",
      "title": "<название>",
      "resolved_by": "<что появилось в данных что закрыло вопрос>"
    }
  ],
  "disclaimer": "Аналитические паттерны из документов. Не являются диагнозом. Все пункты — поводы для обсуждения с врачом."
}
```

## Шаг 5 — После записи

Выведи в чат краткое резюме:
- сколько новых находок
- сколько закрыто с прошлого раза
- главный вывод одной фразой
- путь к файлу
