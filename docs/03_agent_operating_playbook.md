# Agent Operating Playbook

## 1. Команда запуска

Базово:
- `обработай inbox`

С контекстом:
- `обработай inbox`
- `контекст: <краткое пояснение>`

## 2. Цикл обработки

1. Агент смотрит папку `data/raw/inbox/<batch_id>`.
2. Берет первые 3-5 файлов и делает первичную типизацию.
3. Показывает findings и согласует подход.
4. Извлекает факты по всему batch.
5. Обновляет:
   - `data/canonical/documents` (реестр),
   - доменные факты в `data/canonical/*`.
6. Перемещает исходники в:
   - `recognized` / `needs_review` / `non_medical`.
7. Пишет сводку в чат + обновляет `data/derived/reports`.

### 2.1 Режим распознавания (обязательно)

1. Сначала пробовать `text_layer`.
2. Если слой пустой/нечитабельный или качество извлечения не позволяет уверенно извлечь факты, переключаться на `multimodal_vision`.
3. В `documents` и report-артефактах фиксировать фактический режим (`text_layer`, `multimodal_vision` или `hybrid`).
4. Если после fallback уверенность остается низкой, ставить `needs_review`.

## 3. Что обязательно в ответе агента

- сколько файлов обработано,
- какие типы документов найдены,
- какие факты добавлены,
- что требует проверки,
- какие следующие шаги предлагаются.

## 4. Критерий качества

- Пользователь понимает, что произошло, без чтения технических логов.
- Каждый ключевой факт можно связать с исходным файлом.
