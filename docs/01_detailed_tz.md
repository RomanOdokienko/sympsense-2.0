# Детальное ТЗ (текущая версия)

## 1. Что делаем

Ты кладешь пакет документов в папку batch.
Агент:
1. читает документы с диска,
2. делает первичную типизацию,
3. извлекает факты,
4. обновляет структурированную базу,
5. показывает, что изменилось.

## 2. Вход

- Папка: `data/raw/inbox/<batch_id>/`
- Опционально: `context.txt` рядом с файлами.

Рекомендуемый формат `context.txt`:
- цель пакета,
- период,
- что особенно важно извлечь.

## 3. Итерационный порядок работы

### Шаг 1

Сначала агент обрабатывает 3-5 файлов и дает findings:
- предложенные типы документов,
- что извлеклось хорошо,
- что извлеклось с низкой уверенностью.

### Шаг 2

Согласуем реестр документов и представление в UI:
- единый `documents` реестр,
- фильтры по типу/дате/статусу,
- понятные статусы обработки.

### Шаг 3

Согласуем системное извлечение с преемственностью:
- `Document Envelope` (универсальные поля),
- `Domain Facts` (анализы/заключения/назначения/симптомы/рекомендации),
- версии правил извлечения, чтобы новые форматы не ломали старые.

## 4. Статусы документа

`new -> typed -> extracted -> needs_review -> confirmed`

## 5. Ограничения

- Без клинической диагностики и назначений лечения.
- Только локальное хранение данных.
- Любой вывод агента должен быть привязан к источнику.
